1️⃣ How Your Usage Translates to Cost

Every time you use Copilot Chat, Agent mode, or CLI, three things happen:

Token Types

  • Input tokens — your prompt + file context sent to the model
  • Output tokens — the response generated (most expensive)
  • Cached tokens — context reused from earlier (90% cheaper)

Think of it like a phone call

  • 📞 Input = how long you talk
  • 📞 Output = how long the other person talks (costs more)
  • 📞 Cached = "as I mentioned earlier..." (almost free)
(Input tokens × input price) + (Output tokens × output price) + (Cached tokens × cached price) = AI credits consumed

1 AI credit = $0.01 USD
Your goal: Get the answer you need with fewer tokens IN and fewer tokens OUT.

3️⃣ Model Selection — Rules of Thumb

🟢 Use Lightweight (nano, mini, Flash)

  • "Explain this function"
  • "Write a unit test for this"
  • "Fix this typo / syntax error"
  • "Generate boilerplate code"
  • "What does this regex do?"
  • "Add a null check here"

~80% of daily tasks

🔴 Use Powerful (Opus, GPT-5.5) — rarely

  • Complex architecture decisions
  • Multi-file refactoring with tricky logic
  • Debugging concurrency / race conditions
  • Novel algorithm design
  • Security vulnerability analysis

~5% of daily tasks

💡 Tip #2: Start with a lightweight model. Only upgrade if the answer isn't good enough. You can always escalate — you can't un-spend credits.

5️⃣ Prompting Techniques That Save Tokens

A. Be specific (reduces output tokens by 50-80%)

❌ Vague prompt

"Write me a date utility"

→ Model generates 200 lines you don't need

→ High output tokens

✅ Specific prompt

"Write a function formatDate(iso: string): string that returns DD/MM/YYYY format"

→ Model generates exactly what you need

→ 80% fewer output tokens

B. Ask for what you need, not everything

❌

"Explain this entire file"

✅

"Explain what the processPayment function on line 45 does"

Savings: 70-90% fewer output tokens

8

5️⃣ Prompting Techniques (continued)

C. Use constraints

✅ "Fix this bug. Only show me the changed lines."
✅ "Answer in 3 bullet points max."
✅ "No explanations, just the code."

Less output = fewer tokens = less cost.

D. Don't repeat yourself

❌

Pasting the same code block in every message

✅

Reference once, then say "In the function I shared above..."

Cached = 90% cheaper than re-sending

E. Break complex tasks into steps

❌ "Build me a complete REST API with auth, validation, error handling, tests, and docs"
→ Model generates 500+ lines, half wrong, you regenerate = 2x cost
✅ Step by step: 1) Route handler → 2) Validation → 3) Error handling → 4) Test
→ Each step is correct. No wasted regenerations.

7️⃣ Context Management — Less Input = Less Cost

Every open file + conversation history = input tokens sent to the model

✅ Do

  • Close files you're not working on
  • Use @file to reference specific files
  • Start a new chat when switching topics
  • Delete unhelpful messages (click X)
  • Stay in same thread for follow-ups (cached = cheap)

❌ Don't

  • Keep 20 tabs open "just in case"
  • Use @workspace for a single-file question
  • Keep one mega-thread going all day
  • Re-paste code that's already in the thread
  • Include entire files when you only need one function
💡 Tip #3: A focused 3-message thread costs less than a 30-message thread asking the same thing.
@workspace (sends all files) vs. @file:src/auth/login.ts (sends one file)

Difference: 60-80% fewer input tokens

8️⃣ Agent Mode — Deep Dive

What is agent mode?

Agent mode lets Copilot work autonomously: it reads files, runs commands, edits multiple files, and iterates until the task is done.

Why it's expensive

Regular ChatAgent Mode
Model calls1 per message5-20+ per task
ControlYou control each stepAgent decides autonomously
Cost predictabilityPredictableVaries widely
Typical cost~1-5 credits per question~20-200+ credits per task
Each "step" the agent takes = a new model call with full context.
A 15-step agent session on Opus can cost 100+ credits in minutes.

8️⃣ Agent Mode — When to Use / When to Avoid

✅ When TO use agent mode

  • Multi-file refactoring
    ("rename this interface and update all 15 files")
  • Scaffolding
    ("create a new service with controller, repo, and tests")
  • Complex debugging
    ("find the race condition in this test")
  • Tasks requiring reading many files to understand context

❌ When NOT to use agent mode

  • Simple questions
    ("what does this function do?") → use Chat
  • Single-file edits
    ("add a null check") → use Ctrl+I
  • Writing one function → Tab completion or Chat
  • Anything doable in 1-2 chat messages
💡 Ask yourself: "Can I do this in 2-3 chat messages?" If yes, skip agent mode.

8️⃣ Agent Mode — Best Practices to Reduce Cost

1. Be extremely specific about scope

❌ "Improve error handling in this project"

→ Agent explores every file

✅ "Add try/catch to the 3 API functions in src/services/userService.ts"

→ Agent targets exactly what's needed

2. Constrain the files

✅ "Only modify files in src/auth/"
✅ "Don't create new files — only edit existing ones"
✅ "Don't modify tests — I'll update those separately"

3. Provide the plan — don't let the agent plan

✅ "Do these 3 things: 1) Add validation to createUser 2) Add rate limiter to router 3) Add logging to catch blocks"
→ When YOU define steps, the agent executes instead of exploring.

4. Use a versatile model (not Opus) for agent mode

10 calls × Opus = 50x the cost of 10 calls × nano. Use Sonnet/GPT-5.4 for agent — good balance.

5. Stop the agent if it goes off track

Cancel and rephrase with tighter scope. Every confused step = wasted credits.

🔟 Common Questions

Q: Will I get blocked if I use too much?

Your admin has set a monthly budget per user. If you hit it, Chat/Agent stop until next month. Code completions still work — always free.

Q: Can I see how much I've used?

The Copilot status panel shows "Credits: Included with your organization's plan." Per-interaction cost isn't shown live — but you will be notified as you approach your budget limit.

Q: Does it cost more in one language vs. another?

Not directly — cost is token-based. Verbose languages (Java) produce slightly more tokens than concise ones (Python). Small difference in practice.

Q: Do code completions ever cost credits?

No. Never. Always free and unlimited.

Q: Should I avoid agent mode entirely?

No — it's incredibly productive for the right tasks. Use it intentionally, with specific scope, on a versatile model.

Q: What happens when my budget is exhausted?

Chat and Agent stop working. Code completions and inline suggestions continue. Resets at the next billing cycle.

4.  📁  Manage your context — close files, fresh threads, @file not @workspace
